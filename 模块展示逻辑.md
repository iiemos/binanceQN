# 用户端功能模块及显示逻辑说明

## 概述
用户端面向已连接钱包的普通用户、VIP用户、代理/合伙人。所有功能根据用户状态（是否有效用户、VIP等级、是否代理/合伙人）动态显示。

---

## 模块列表

### 1. 钱包连接与用户初始化
- **显示条件**：始终显示，未连接时显示“连接钱包”按钮。
- **功能**：调用钱包插件（MetaMask等）获取地址，后端查询用户信息（等级、是否有效用户、UID绑定状态等），前端存储用户状态。
- **交互**：连接成功后，根据用户是否已购买权限（有效用户）跳转至购买页或仪表盘。

---

### 2. 权限购买模块
- **显示条件**：用户已连接钱包但未购买权限（非有效用户）。
- **功能**：
  - 显示推荐人地址输入框（URL参数 `ref` 自动填充）。
  - 购买按钮：调用合约 `purchaseAndBind(inviter)`，支付100 USDT。
  - 交易确认后，后端分配跟单码，前端展示弹窗提示用户保存。
- **显示逻辑**：若用户已购买权限，则隐藏此模块，显示仪表盘。

---

### 3. 仪表盘核心模块（仅对有效用户显示）

#### 3.1 用户等级徽章
- **显示条件**：始终显示在仪表盘顶部。
- **内容**：显示当前等级（有效用户/VIP1~VIP5），若为代理/合伙人则额外显示“代理权限”或“合伙人权限”徽章。

#### 3.2 统计卡片（4个）
- **显示条件**：始终显示。
- **内容**：
  - 直推人数
  - 团队总业绩（伞下所有有效用户及升级费用总额）
  - 累计收益（所有分润收入）
  - 待还金额（仅VIP3/VIP5主动升级且分期未还清时显示，否则显示0）
- **数据来源**：后端接口 `/user/info` 返回。

#### 3.3 跟单码展示与复制
- **显示条件**：用户已购买权限（有效用户）。
- **内容**：显示已分配的跟单码，提供复制按钮。
- **说明**：若用户通过代理/合伙人配置的个人码购买，则显示代理的码；否则显示系统分配的码。

#### 3.4 升级入口
- **显示条件**：根据用户当前等级和条件动态显示多个升级按钮，可能同时显示多个（如同时满足VIP1晋升和付费升级）。
- **升级类型**：
  - **免费晋升（VIP1）**：当用户满足 `直推有效用户数 >= 5` 且当前等级 < VIP1 时显示“晋升 VIP1”按钮。
  - **免费晋升（VIP2~VIP5）**：当用户满足 `伞下任意3个独立市场线出现前一级别` 且当前等级低于目标等级时，显示对应晋升按钮（如“晋升 VIP2”）。条件由后端晋升引擎计算，前端通过接口获取可晋升等级列表。
  - **付费升级（VIP3）**：当用户当前等级 < VIP3 且未达到免费晋升条件时，显示“升级 VIP3（代理）”，提示支付600 USDT首付。按钮点击后调用合约 `upgradeLevel(4)` 支付600 USDT。
  - **付费升级（VIP5）**：当用户当前等级 < VIP5 且未达到免费晋升条件时，显示“升级 VIP5（合伙人）”，提示支付2000 USDT首付。按钮点击后调用合约 `upgradeLevel(6)` 支付2000 USDT。
- **交互**：点击按钮后弹出确认框，调用合约方法，等待交易确认，刷新用户信息。

#### 3.5 团队结构图
- **显示条件**：用户为有效用户或VIP。
- **功能**：以树形图展示用户伞下前几层（如深度2~3）的下级，支持展开查看更多（通过分页或懒加载）。点击节点可查看该用户的简要信息（等级、直推数等）。
- **数据**：调用 `/user/tree?depth=2` 获取。

#### 3.6 收益明细表
- **显示条件**：始终显示。
- **功能**：表格展示分润流水，包含时间、类型（直推/极差/代理收益）、金额。支持按类型筛选。
- **数据**：调用 `/commission/list` 分页获取。

#### 3.7 绑定币安UID
- **显示条件**：用户未绑定UID时显示表单。
- **功能**：输入UID，调用 `/user/bind-uid` 绑定。绑定后显示已绑定的UID，不可修改（或可修改需额外权限）。

---

### 4. 代理/合伙人专属模块（仅当用户为代理或合伙人时显示）
这些模块可放在仪表盘内单独区域，或通过独立标签页（如“代理后台”）显示。

#### 4.1 跟单码配置（代理/合伙人）
- **显示条件**：用户为代理或合伙人（`isAgent = true` 或 `isPartner = true`）。
- **功能**：表单录入/修改个人币安私域跟单码，保存后供下级用户领取。提供当前配置的码展示。
- **交互**：输入框，保存按钮，调用 `/agent/code` 接口。

#### 4.2 代单收益提取
- **显示条件**：用户为代理或合伙人。
- **功能**：
  - 显示当前可提取的收益总额（从币安代单账户获取的10%部分）。
  - 输入提取金额（自动限制不超过可提取总额），点击提取。
  - 后端处理分发（直推、极差奖励发放），前端展示结果。
- **数据**：调用 `/agent/earnings` 获取可提取额；提取调用 `/agent/withdraw`。

#### 4.3 团队业绩统计（代理/合伙人）
- **显示条件**：用户为代理或合伙人。
- **功能**：展示伞下总业绩、各级别人数（VIP1、VIP2等）、近期收益趋势图。
- **数据**：调用 `/agent/team` 接口。

## 交互流程说明

1. **首次连接**：用户连接钱包，前端调用 `/user/info`，若返回 `isActive: false`，显示购买页；否则显示仪表盘。
2. **购买权限**：用户在购买页输入推荐人地址（可选），点击购买，调用合约 `purchaseAndBind`，监听交易回执。成功后前端轮询获取跟单码（或通过WebSocket），弹窗展示。
3. **仪表盘刷新**：用户任何操作（升级、购买等）后，前端应重新调用 `/user/info` 更新状态，并根据新状态调整显示模块。
4. **升级操作**：点击升级按钮，调用合约对应方法，等待确认，刷新用户信息。对于付费升级，需传入对应金额。
5. **代理/合伙人后台**：若用户是代理/合伙人，可在仪表盘内切换标签页或显示额外卡片，执行配置、提取等操作。

---

## 开发注意事项
- 所有显示条件必须基于后端返回的最新数据，不能依赖前端本地缓存。
- 升级按钮的显示应同时考虑免费晋升和付费升级两种可能，不能相互覆盖。
- 分期还款信息（待还金额）应实时展示，由后端返回。
- 网体结构图需支持递归或分页加载，避免一次性加载过多节点导致性能问题。

此文档可直接用于前端开发人员实现用户端功能模块的显示逻辑。
